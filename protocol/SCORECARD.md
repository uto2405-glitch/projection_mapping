# SCORECARD v0.1 — 라이브 드로잉 프로젝션

> 이 문서가 "합격"의 유일한 정의다. 에이전트는 이 문서를 수정할 수 없다.
> 개정은 사람(영택)의 승인을 거쳐 개정 이력에 기록된 경우에만 유효하다.

## 목적문 (0단계 승인본)

아이패드 사파리에서 펜으로 긋는 획이, 프로젝터에 연결된 출력 브라우저에 즉시 나타나고,
글로우·잔상 연출이 함께 얹히며, 출력은 4코너 워프(기본) 또는 N×M 격자 워프
(곡면 손정렬, 개정 2호)로 실물 표면 한 면에 정렬되는
웹(WebGL) 라이브 드로잉 프로젝션 도구. 같은 기기(창 두 개)와 두 기기(네트워크 페어)
구성을 하나의 동기화 추상 계층으로 지원한다. 채점은 2단 — 책상 실험 합격 후
실제 프로젝터·현장 조건으로 승격한다.

## 표준 부하 시나리오 (모든 자동판정의 공통 조건)

- 기록된 획 세트 재생: 초당 3획, 총 200획, 점~긴 곡선 혼합 (grader/scenario.mjs, 시드 고정)
- 잔상·글로우 켠 상태
- 채점기(Playwright)가 앱을 띄워 시나리오를 주입하고 바깥에서 관측한다

## A. 자동판정 — 코드가 pass/fail 선언

| ID | 기준 | 측정 |
|----|------|------|
| A1 | 같은 기기 획 지연 중앙값 ≤ 50ms | 입력 주입 시각 → 출력 렌더 마크, 픽셀 진실 검사로 교차검증 |
| A2 | 두 기기 획 지연 중앙값 ≤ 120ms | 실기기 항목 — 승격 시 왕복÷2 근사 측정 (책상에선 PENDING) |
| A3 | 출력 평균 fps ≥ 55 | 채점기가 출력 페이지에 자체 rAF 카운터 주입 (표준 시나리오 5분) |
| A4 | 5분 세션 크래시·콘솔 에러 0 | pageerror / console.error 수집 |
| A5 | 연속 100획 전송 유실 0 | 책상: 창 2개 모의 / 승격: 실기기 페어 |
| A6 | 4코너 정렬값 새로고침 후 복원 일치 | 저장 → reload → 값 비교 |
| A7 | PNG 내보내기 동작, 캔버스 내용 포함 | 다운로드 이벤트 + PNG 시그니처 + 비어있지 않음 |
| A8 | 출력 캔버스 백킹 해상도 ≥ 1920×1080 | canvas width/height 검사 (헤드리스에서 전체화면 대용) |
| A9 | 획 충실도: 렌더 포인트 ≥ 입력 포인트의 90% | 주입 포인트 수 vs 레지스트리 렌더 포인트 수 |

## B. 사람 시사 — 영택만 판정 가능

| ID | 질문 |
|----|------|
| B1 | 손맛: 아이패드 실기기에서 획이 "스윽" 따라오는가 |
| B2 | 연출의 예쁨: 글로우·잔상이 프로젝터 투사에서 실제로 근사한가 (취향 앵커 참조) |
| B3 | 획 품질: 워프(4코너·격자 곡면) 적용 후에도 곡선이 부드러운가 |
| B4 | 정렬 UX(승격 시): 4코너 세팅이 헤매지 않고 3분 이내에 끝나는가 |

시사에서 발견된 결함은 개정 이력을 통해 채점표 조항으로 승격한다.

## C. 금지 — 위반 즉시 불합격

| ID | 조항 |
|----|------|
| C1 | 외부 클라우드 의존 금지 — LAN 안에서 완결 (전시장 인터넷 불안정 대비) |
| C2 | 테스트·채점기 코드 수정 금지 / 이 채점표 수정 금지 (개정은 사람 승인 경유만) |
| C3 | 제외 명시 목록 선제 구현 금지: 멀티 서페이스(여러 면·멀티 프로젝터) · 자동 캘리브레이션 · 압력 감지 굵기 · 레이어/이미지 불러오기 · 다중 사용자 동시 드로잉 — 단, 굴곡면 "한 면"의 N×M 격자 손정렬은 개정 2호로 허용 |
| C4 | 앱의 자가 보고 지표를 채점 근거로 사용 금지 — 채점기는 외부 관측 우선, 자가 보고는 픽셀 진실 검사로 교차검증 |

## D. 굿하트 점검 — 지표를 속이는 경로와 봉쇄

| ID | 속임 경로 | 봉쇄 |
|----|-----------|------|
| D1 | 지연 단축을 위해 포인트를 과도하게 솎음 | A9 충실도 + B3 시사 |
| D2 | fps를 위해 효과를 몰래 끄거나 저품질화 | 채점기가 토글 상태 검증 + 스크린샷 글로우 픽셀 확산 검사 |
| D3 | 유실 0을 위해 획을 배치로 묶어 지연 증가 | A1·A2와 A5의 동시 충족 요구 |
| D4 | 채점기·채점표 자체를 고쳐 통과 | C2 + .claude/settings.json의 Edit/Write deny (물리 봉쇄) |

## 판정 구조

- 실험 합격 = A1, A3~A9 전부 PASS + B1~B3 전부 합격
- 현장 승격 = 실험 합격 + A2 PASS + 4코너 정렬 소요 ≤ 3분 + 정렬 후 30분 드리프트 없음 + B4 합격

## 기술 스택 (0단계 승인)

Three.js + Vite + Node ws 릴레이(출력 PC에서 실행, 아이패드는 QR로 접속),
같은 기기는 BroadcastChannel. 렌더 목표 1080p.
