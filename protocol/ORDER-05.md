# 발주서 #5 — 격자 워프(곡면 손정렬) + 브라우저 시뮬레이션 환경 (2026-07-17 발주)

발주 근거: SCORECARD 개정 2호(격자 워프, 사람 승인 "격자 워프만") + 사람 포괄 승인
("모든 구현을 승인" 2026-07-17 — 이후 확장도 발주서 단위로 집행) + 지시
"프로젝터가 없으니 브라우저 환경에서 모두 테스트 가능하게".

## 범위

1. **N×M 격자 워프** — 4코너(호모그래피)의 일반화. 3×3/4×4/5×5 제어점을
   아이패드에서 드래그해 굴곡면 한 면에 손정렬 (Catmull-Rom 텐서 보간 → 정점 메시).
   - 4코너 모드 ↔ 격자 모드 전환 (전환 시 현재 워프를 샘플링해 연속 초기화)
   - 저장: `ldp:warp` (격자) — 기존 `ldp:corners`(A6 계약)는 4코너 모드로 불변 보존
   - 동기화: `{t:'warp'}` 메시지 + `warp-req` 상태 요청 (기존 corners 경로 하위호환)
2. **가상 표면 시뮬레이터** — 출력 URL `?sim=curtain|column|globe`:
   절차적 굴곡면(커튼 주름·원기둥·지구본)을 배경으로 깔고 출력 캔버스를
   screen 블렌드로 합성 — 프로젝터의 가산 광학을 브라우저에서 재현.
   채점기는 sim 파라미터를 쓰지 않으므로 계약 무영향.
3. **랜딩 허브** — 드로잉·출력·시뮬레이션 3종 바로가기.

## 합격 목표

- 공식 `npm run grade` 전항목 유지 (4코너 기본 모드 회귀 금지)
- 신규 모의 시사: S19 격자 워프(보간 수치 대조·저장·모드 복귀) 통과
- 시각 검증: 가상 굴곡면 위 격자 정렬 스크린샷

## 금지

- C1·C2·C4 전부. C3 잔여 목록(멀티 서페이스=여러 면·자동 캘리브레이션·압력 굵기·
  레이어/이미지·다중 동시 드로잉) 접근 금지.
