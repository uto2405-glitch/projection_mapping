# 발주서 #4(=#3.5) — 시사·현장 대비 마감 고도화 (2026-07-17 발주)

발주 근거: 사람 포괄 승인(2026-07-17, "이대로 추천 방향으로 쭉 진행·고도화").
로드맵 #4(미디어 레이어)가 아니라 기존 범위의 마감 품질 사이클 — C3 제외 목록과
채점표는 건드리지 않는다. B1(손맛)·B3(획 품질)·B4(정렬 UX)·현장 운영성에 직결되는
항목만 담는다.

## 범위

1. **획 스무딩** — 입력 폴리라인을 중점 이차곡선으로 렌더 (B3: 곡선 부드러움).
   pointsRendered 의미(입력 포인트당 1, 누적-실렌더)와 렌더 마크 시점은 불변.
2. **실행취소** — 마지막 획 취소(`[data-test="undo"]`), 동기화 계층으로 출력에도
   반영. 벡터 재드로우 기반이라 지우개 획 취소 시 지워진 획이 복원된다.
   레지스트리에서 해당 획 레코드 제거(실제 렌더 상태 반영, C4 정합).
3. **펜·연출 설정 영구화** — 색·굵기·연출 상태를 draw 쪽 localStorage(`ldp:pen`)에
   저장·복원 (리로드·재방문 시 이어서).
4. **동기화 상태 배지** — draw 툴바에 로컬/릴레이 연결 상태 표시 (현장 진단).
5. **정렬 미세조정** — 핸들 드래그를 상대 이동으로 바꾸고 미세 모드(×0.25) 추가
   (`[data-test="align-fine"]`) — B4: 3분 내 정렬의 마지막 1cm.
6. **현장 기동 안정화** — start-field가 빌드+프리뷰(`npm run field`)로 서빙
   (개발 서버의 파일 감시·HMR 재로드 없는 장기 운영). 채점 경로(`npm run dev`,
   포트 5173)는 계약 그대로 유지.

## 합격 목표

- 공식 `npm run grade` 전항목 유지 (회귀 금지)
- 모의 시사(bench/scenarios.mjs) 전항목 + 신규 시나리오(undo·설정 복원·스무딩) 통과
- 신규 코드 적대 감사에서 생존 발견 0건이 될 때까지 수정

- 채점 계약 셀렉터·의미 변경 금지 (추가만 허용).
